What is a skill?

A skill is a folder Claude reads automatically when your request matches it. Think of it as a recipe card for a task you do often — you write the instructions once, and Claude follows them every time.

You say "make me a deck"
Claude loads the skill
Polished HTML, your way
What makes a good skill

Four marks of a skill worth keeping

01

A sharp description

Claude reads it to decide when to run the skill. Vague description — the skill never fires.

02

One clear job

A skill that does one thing well beats one that tries to do everything. Narrow wins.

03

Reference files

Your style, your rules, your examples — kept in files the skill loads only when needed.

04

It improves

A learnings.md file captures what you correct, so the skill gets better with use.

What makes a good skill

The description is the trigger

It is not a summary for humans. It is how Claude decides to reach for the skill — so name the exact things a user would say.

Too vague
"Helps with slides."
Claude can't tell when this applies. The skill sits there and never fires.
Sharp
"Use when the user says 'make a deck', 'build slides', 'visualize this'…"
Names real trigger phrases. Claude knows exactly when to load it.

The skill

Meet the visualization skill

You've just seen what makes a skill good. This one is built exactly that way — and it's ready for you to download and use. It turns a request into a polished, single-file HTML artifact, in any of three formats:

Slides

Presentations and training decks. Arrow-key navigation, progress bar, speaker notes.

Dashboard

Tabbed, multi-view reference. For exploring something at your own pace.

Diagram

One page, one concept. A flow, a comparison, an explanation.

Set up

Install it

Two commands, typed straight into Claude Code. No downloads, no files to move.

Step 1 — do this once

Add the AI Study Camp skill library to Claude Code:

/plugin marketplace add aistudycamp/skills
Step 2 — install the skill

Then install visualization:

/plugin install visualization@aistudycamp-skills

That's it — installed. Say "make me a deck" and it activates on its own.

Customize

Make it yours

It works right away with a neutral theme. To make it produce your decks, set up your design system — two ways:

Let Claude do it

Point it at a few decks you like, or paste your colors. It writes design-system.md for you.

Hands-on

Edit references/design-system.md yourself — a filled-in example is right there to copy.

Run it

Use it — and watch it learn

Just ask. Say "make a deck about X" or "visualize this" — the skill reads your design system, follows your style, and builds the HTML.

You ask
It builds it your way
You correct something
It remembers — learnings.md

Every correction makes the next deck closer to what you wanted.
