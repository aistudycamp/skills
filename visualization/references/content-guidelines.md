# Content Guidelines

How to keep a visualization clear. The design system makes it *look* good — these rules make it *communicate*.

## Keep text minimal

- Slides are not documents. If a slide has a paragraph, cut it to a sentence. If it has a sentence, cut it to a phrase.
- **One idea per slide.** If a slide needs the word "and", it is probably two slides.
- Lead with the headline. The takeaway goes in the slide title, not buried in a bullet.
- Bullets: about 6 maximum per slide, each one line. A long bullet means it should be its own slide — or be cut.

## Show, don't tell

- A diagram beats a paragraph. A before/after beats a description of the change.
- Use the components in `design-patterns.md` — flow diagrams, compare cards, stat cards, timelines — instead of plain text whenever the content allows.
- Numbers belong in stat cards (big number + short label), not inside sentences.

## Structure

- Open with a title slide. Close with a clear takeaway or next-step slide.
- Give the deck a spine — each slide should follow logically from the one before it.
- Use small uppercase section labels so the audience always knows where they are.

## Polish

- Generous whitespace. Cramped slides read as unfinished.
- Consistent color meaning — if green meant "done" on slide 3, it means "done" on slide 9.
- For presenter decks, put detail in speaker notes (toggled with the `n` key) and keep the slide itself clean.
